Detalles del Taller

Esta sección contiene una explicación detallada del taller, incluyendo el progreso y los requerimientos.

Es un proyecto de desarrollo de software basado en Java, utilizando Maven para la gestión de dependencias y construcción del proyecto. Este proyecto incluye la configuración para desplegar la aplicación en contenedores Docker, lo cual indica una arquitectura orientada a microservicios o despliegues escalables. Además, la presencia de archivos relacionados con Kubernetes sugiere que el proyecto está diseñado para ser desplegado y gestionado en un entorno de orquestación de contenedores, lo que permite una mayor escalabilidad y eficiencia en la gestión de recursos. La estructura del proyecto y los archivos incluidos sugieren un enfoque en prácticas modernas de desarrollo y despliegue de aplicaciones, así como una preocupación por la escalabilidad y la gestión eficiente de la infraestructura.

Requisitos Funcionales

    - Presentación del Grupo:

    Página dedicada a mostrar el equipo.

    - Formulario de Contacto

    Campos para nombre, correo electrónico, mensaje y un botón de envío.

    Validación de todos los campos en el lado del cliente para verificar que no estén vacíos.

    Validación adicional en el correo electrónico para asegurar un formato correcto.

    Mensaje de confirmación o error después del envío del formulario.

    - Conexión con la Base de Datos:

    Almacenamiento de las consultas del formulario en la base de datos proporcionada por el profesor.

    Garantizar la seguridad de los datos mediante la sanitización de entradas para prevenir inyecciones SQL.

    - Sección Wiki:

    Descripción detallada del proceso de desarrollo, tecnologías utilizadas y desafíos encontrados.

    Incluir enlaces a recursos o documentación relevante.

    - Página de Agradecimientos:

    Espacio para agradecer a los colaboradores y participantes.

    - Navegación:

    Un menú de navegación eficiente y fácil de usar que conecte todas las secciones de la página.

    Requisitos No Funcionales

    - Usabilidad:

    Diseño intuitivo y fácil de usar, adecuado para todos los usuarios, independientemente de su experiencia técnica.

    Compatibilidad con diferentes tamaños de pantalla y dispositivos (responsive design).

    - Rendimiento

    Tiempos de carga rápidos para todas las páginas y recursos.

    Optimización de imágenes y scripts para mejorar la velocidad.

    - Seguridad:

    Implementar medidas de seguridad para proteger la página de ataques comunes como XSS y CSRF.

    Asegurar la transmisión de datos del formulario mediante HTTPS.

    - Accesibilidad

    Cumplir con las pautas de accesibilidad web WCAG para asegurar que la página sea accesible para personas con discapacidades.

    - Despliegue y Mantenimiento:

    La página debe estar correctamente desplegada en una plataforma accesible públicamente.

    Documentación adecuada para el mantenimiento y actualizaciones futuras.
